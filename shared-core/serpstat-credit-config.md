# Serpstat Credits 预算控制系统

## 概览

本文档定义 Serpstat API 信用点消耗的所有可调参数。根据月度预算档位调整这些参数。

---

## 信用预算档位

| 档位 | 月度Credits | 适用场景 |
|------|------------|---------|
| 入门档 | 50,000 | 单产品线，1-2个区域 |
| 成长档 | 100,000 | 2-3条产品线，多区域 |
| 专业档 | 200,000 | 全品类运营（默认） |
| 企业档 | 500,000 | 代理/多站点网络规模 |

---

## 可调参数表

| 参数名 | 默认值 | 可调范围 | 说明 | 对Credits影响 |
|--------|--------|---------|------|--------------|
| `SERPSTAT_MONTHLY_LIMIT` | 200000 | 50000-500000 | 月度信用上限 | 硬性封顶 |
| `SEED_KEYWORD_COUNT` | 20 | 10-50 | 每产品线核心种子词数 | 线性：每词+50 |
| `SEED_EXPANSION_DEPTH` | 50 | 20-100 | 每种子词拓展的相关词数 | 线性：每结果+1 |
| `COMPETITOR_DOMAIN_COUNT` | 5 | 3-8 | 重点监控竞品域名数 | 线性：每域名~600 |
| `PAGES_PER_COMPETITOR` | 3 | 1-5 | Serpstat分页深度 | 线性：每页~100 |
| `RESULTS_PER_PAGE` | 100 | 50-1000 | 单次API返回条数(size参数) | 影响数据密度 |
| `TARGET_REGIONS_COUNT` | 2 | 1-4 | 目标区域数 US/EU/CN/APAC | 乘数：每区域x倍 |
| `VOLUME_UPDATE_BATCH` | 500 | 200-2000 | 月度核心词刷新量 | 线性：每词x2 |
| `INCREMENTAL_CACHE_TTL` | 30 | 7-90 | 缓存天数（到期后重新获取） | 越高=调用越少 |
| `MAX_ROWS_PER_REQUEST` | 1000 | 100-60000 | 单次API调用最大行数 | 安全上限8000 |
| `COMPETITOR_KEYWORDS_PER_DOMAIN` | 250 | 100-500 | 每域名最大采集关键词数 | 线性增加 |
| `KEYWORD_EXPANSION_MULTIPLIER` | 2.5 | 1.5-5 | 种子词:拓展词比例 | 控制总词库规模 |
| `ZERO_VOLUME_WHITELIST_COUNT` | 50 | 20-100 | 零搜索量技术参数保护词数 | 影响极小 |
| `RANK_TRACK_KEYWORDS` | 50 | 10-200 | SERP排名追踪关键词数 | ~1 credit/词 |
| `SITE_AUDIT_PAGES` | 1000 | 100-10000 | 技术审计扫描页面数 | ~1 credit/百页 |
| `CLUSTER_ANALYSIS_BATCH` | 200 | 50-500 | 单次聚类分析关键词数 | ~5 credits/词 |

## 月度任务预算模板

调整参数前，用以下公式预估月度消耗：

| 任务 | 计算公式 | 默认消耗 | 频率 |
|------|---------|---------|------|
| 种子词拓展 | `种子词数 x 拓展深度 x 区域数 x 2` | 2000 | 月度 |
| 一级竞品关键词 | `竞品数 x 页数 x 100 x 区域数` | 3000 | 月度 |
| 二级竞品关键词 | `3 x 页数 x 100 x 1` | 900 | 季度 |
| 搜索量刷新 | `刷新批次 x 区域数 x 2` | 2000 | 月度 |
| Gap分析 | `100 x 区域数 x 2` | 400 | 月度 |
| 网站审计 | `审计页面数` | 1000 | 月度 |
| 排名追踪 | `追踪词数 x 区域数` | 100 | 月度 |
| 聚类分析 | `聚类批次 x 5` | 1000 | 季度 |
| 手动上传处理 | `100 x 区域数` | 200 | 月度 |

**默认月度总计**：约 8,000-10,000 credits（占200K档位的5%）

---

## 省Credits策略

1. **延长缓存周期**（`INCREMENTAL_CACHE_TTL` 30->60天）：刷新调用省约30%
2. **减少竞品分页深度**（`PAGES_PER_COMPETITOR` 3->1）：竞品成本降66%
3. **缩减拓展深度**（`SEED_EXPANSION_DEPTH` 50->25）：种子研究成本减半
4. **二级竞品改季度扫描**：从月度改为季度
5. **批量操作**：始终使用 Batch Analysis 替代单个调用

## 安全限制

| 限制项 | 数值 | 用途 |
|--------|------|------|
| `SINGLE_CALL_LIMIT` | 8000 | 防止单次调用超限 |
| `DAILY_QUERY_LIMIT` | 500 | Team套餐日查询上限 |
| `API_RATE_LIMIT_RPS` | 1 | 每秒请求数节流 |
| `MAX_RESULTS_PER_REQUEST` | 60000 | API绝对上限 |
